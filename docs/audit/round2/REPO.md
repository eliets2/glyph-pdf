## Secret history (B-01) — CRITICAL RELEASE BLOCKER

### Status: KEYS ARE PRESENT IN BOTH HISTORY AND THE CURRENT INDEX

**All three credential files appear in commit `a6ea6aa` and are still tracked in the working tree:**

```
tests/fixtures/signing/ca.key      — PKCS#8 RSA private key (confirmed real: "-----BEGIN PRIVATE KEY-----")
tests/fixtures/signing/signer.key  — private key (same commit)
tests/fixtures/signing/test_signer.p12 — PKCS#12 bundle (confirmed binary crypto material, ~3.3 KB)
```

The `.gitignore` was updated (commit `b0b12fb`) to exclude `tests/fixtures/signing/*.key`, `*.p12`, etc., but **`.gitignore` only prevents future commits — it does NOT remove files already tracked**. The files remain in the index and in the full history across 211 commits.

### Why this is a release blocker

Even though these are test/self-signed credentials (not production keys), making the repo public with private keys in history:
- Triggers GitHub's secret scanning and will flag the push.
- Leaks the test CA private key, enabling an attacker to sign PDFs with a chain that test suites trust.
- Sets a bad precedent for a security-focused PDF tool.

### Required action before any public push

1. **Remove the files from the current index first:**
   ```
   git rm --cached tests/fixtures/signing/ca.key \
                   tests/fixtures/signing/signer.key \
                   tests/fixtures/signing/test_signer.p12
   git commit -m "chore: untrack signing key fixtures (gitignored, regenerated by script)"
   ```

2. **Purge from entire history using `git filter-repo` (SP-5b):**
   ```
   pip install git-filter-repo
   git filter-repo --path tests/fixtures/signing/ca.key --invert-paths
   git filter-repo --path tests/fixtures/signing/signer.key --invert-paths
   git filter-repo --path tests/fixtures/signing/test_signer.p12 --invert-paths
   ```
   Or in one pass:
   ```
   git filter-repo \
     --path tests/fixtures/signing/ca.key --path tests/fixtures/signing/signer.key \
     --path tests/fixtures/signing/test_signer.p12 \
     --invert-paths
   ```

3. **Verify** with `git log --all --oneline -- tests/fixtures/signing/ca.key` — must return empty.

4. **Force-push** all branches to origin after the purge (history rewrite affects every branch).

5. **Rotate** the test certificates — regenerate via the signing fixture script so the old material is invalidated even if it leaked.

> **Note:** `git filter-repo` rewrites all commit SHAs. Coordinate with any collaborators and update CI/branch-protection rules after the force-push.
