## 1. Executive summary

The codebase is **substantial and largely real** ‚Äî it builds clean, 32/32 tests pass, and many headline
features are genuinely wired (real PAdES trust-policy `CMS_verify`, OCSP-verified-before-DSS, real
content-stream text excision with Edact-Ray glyph-advance + `Tr==3` OCR-scrub, ~20 sanitization vectors,
real Myers diff, real MRC/JBIG2/JP2K pipeline, real multi-provider AI wiring). The license-contamination
guards (MuPDF/Poppler/veraPDF-subprocess/DjVu-off) are correctly enforced.

**But it is not v1.0.0-ready, and several defects are dangerous.** The audit produced **~111 findings**
(11 Critical, 25 High). The pattern that matters most: **security operations that silently fail or lie** ‚Äî
encryption that reports success while writing plaintext, certification that downgrades itself and drops
`/DocMDP`, a valid signature that can render as "forged", a timestamp that writes 4 null bytes and returns
true, and a `ProvenanceGuard` non-negotiable that is **constructed but never called**. Combined with a
broken certificate-encryption path (all-zero AES key) and redaction that doesn't excise image XObjects,
the app can present a green "signed / encrypted / redacted" UI over a document that is none of those things.

## 4. Cross-component catastrophe chains (pre-capstone synthesis)

> The dedicated emergence-engine capstone (`H`) is pending the cap reset; these are the chains already
> visible from A-G and must drive fix sequencing.

- **CHAIN-1 "Silent un-signing of a signed document."** D-05 (ProvenanceGuard never called) + E-08
  (writeUpdate not guarded for signatures) + E-01 (certify downgrade) ‚áí a signed/certified born-PDF can be
  edited and re-saved with an invalidated/absent signature while the UI still presents it as valid. **Highest-leverage break point:** wire ProvenanceGuard (D-05) + enforce incremental writeUpdate (E-08).
- **CHAIN-2 "Confidential-but-plaintext."** A-01 (broken cert encryption) + E-04 (encrypt-failure shows success) ‚áí user ships a file they believe is encrypted that is plaintext or undecryptable.
- **CHAIN-3 "Redacted-but-leaking, and the test is blind."** A-02 (image XObject not excised) + A-05 (fixed redaction box) + G-02 (unrecoverability test greps compressed bytes) ‚áí PII ships **and** the safety test that should catch it cannot.
- **CHAIN-4 "Supply-chain ‚Üí key theft."** B-03 (update downloadUrl unchecked ‚Üí RCE) + B-02 (plaintext key fallback) ‚áí a poisoned update can exfiltrate stored API keys. (Mostly neutralized once cloud AI + update are removed/hardened per WP-1.)
